• <b><a href="https://huggingface.co/papers/2603.23516">MSA: Memory Sparse Attention for Efficient End-to-End Memory Model Scaling to 100M Tokens</a></b>
MSA addresses the fundamental tension between full attention's quadratic cost and existing approximations (linear attention, RNNs, RAG) that either degrade at scale or lack end-to-end differentiability. The approach combines top-k sparse attention with document-wise RoPE and a KV cache compression engine to achieve near-linear complexity, running 100M token contexts on just 2×A800 GPUs. On NIAH and long-context QA benchmarks, MSA shows less than 9% degradation scaling from 16K to 100M tokens — outperforming same-backbone RAG stacks. For RAG and retrieval practitioners, this represents a credible architectural alternative: an end-to-end trainable memory mechanism that doesn't require chunking, indexing, or retrieval pipelines.
<i>Interview angle:</i> Expect probes on the architectural tradeoffs between sparse attention, RAG, and RNN-based memory — and specifically why end-to-end differentiability matters for memory maintenance.

• <b><a href="https://huggingface.co/papers/2603.24755">SlopCodeBench: Benchmarking How Coding Agents Degrade Over Long-Horizon Iterative Tasks</a></b>
SlopCodeBench attacks a blind spot in coding agent evaluation: existing benchmarks measure pass/fail on isolated tasks but ignore how code quality degrades over iterative, multi-step extension. Agents were required to extend their own solutions across multiple steps without prescribed structure — and every tested model failed to complete any problem end-to-end. Agent-generated code was 2.2x more verbose than comparable open-source projects and exhibited faster complexity growth than human code over time. The benchmark ships with 20 problems and is actively soliciting contributions for multi-step coding challenges.
<i>Interview angle:</i> When discussing code generation systems, interviewers may now probe how you'd evaluate long-horizon quality degradation, not just unit test pass rates.

• <b><a href="https://machinelearning.apple.com/research/latent-lookahead">Thinking into the Future: Latent Lookahead Training for Transformers</a></b>
Latent lookahead targets the hard constraint in autoregressive LMs: every token gets exactly one forward pass, regardless of difficulty, and the model cannot explore alternative continuations before committing. The approach inserts multi-step lookahead at selected positions by recursively feeding hidden states back into context for τ steps, each supervised against the next τ ground-truth tokens — investing more compute where it matters. On planning tasks (maze solving, Sudoku, ProsQA), latent lookahead substantially outperforms both autoregressive and non-autoregressive baselines. This is a clean alternative to chain-of-thought that operates entirely in continuous latent space rather than discrete token space.
<i>Interview angle:</i> A sharp interviewer might ask how this compares to chain-of-thought in terms of compute budget, discrete vs. continuous reasoning, and training vs. inference cost.

• <b><a href="https://machinelearning.apple.com/research/exclusive-self-attention">Exclusive Self Attention</a></b>
Exclusive Self Attention (XSA) is a minimal modification to standard self-attention that constrains each token to attend only to information orthogonal to its own value vector — explicitly excluding self-position information from the attention context. The motivation is that standard SA conflates a token's own representation with contextual information, potentially limiting context modeling quality. XSA consistently outperforms standard SA across model sizes up to 2.7B parameters, with gains increasing as sequence length grows — suggesting the fix becomes more impactful at longer contexts.
<i>Interview angle:</i> Be ready to discuss how self-attention aggregates information and why a token attending strongly to itself could crowd out relevant contextual signal.

• <b><a href="https://simonwillison.net/2026/Mar/26/quantization-from-the-ground-up/#atom-everything">Quantization from the Ground Up</a></b>
Sam Rose's interactive essay covers LLM quantization end-to-end: floating point binary representation, block quantization mechanics, outlier values ("super weights"), and how their preservation is handled in real quantization schemes. The key empirical finding, validated with llama.cpp perplexity and GPQA benchmarks on Qwen 3.5 9B: 16-bit to 8-bit carries almost no quality penalty; 16-bit to 4-bit retains roughly 90% of quality depending on metric. The super-weight finding is notable — removing even a single outlier weight can cause a model to output gibberish, which is why production quantization schemes often store these in a separate table.
<i>Interview angle:</i> Interviewers probing model compression will expect you to explain block quantization, the outlier problem, and the practical quality/size tradeoff curve.

<b>Learning Rate Scheduling</b>
Learning rate scheduling is the practice of varying the learning rate during training rather than holding it fixed — starting high to escape poor initializations quickly, then decaying to converge precisely without overshooting loss minima. The intuition: a large LR early covers ground fast; a small LR late makes fine-grained adjustments that a large LR would blow past. In practice, the schedule choice (cosine decay, linear warmup + decay, OneCycle, etc.) can be as impactful as architecture changes, especially for LLMs and fine-tuning runs where gradient dynamics shift significantly between phases. A common misconception is that warmup is optional for small models — in reality, warmup is critical whenever using adaptive optimizers like Adam on transformers, because early gradient estimates are noisy and a large initial LR causes unstable updates that can permanently damage convergence. Cosine annealing with restarts is often preferred in production because it naturally escapes local minima at restart boundaries while still converging.

<b>Q1:</b> Why is a linear warmup phase typically used at the start of LLM training? → <i>Early Adam gradient moment estimates are unreliable, so a large initial LR causes unstable parameter updates; warmup gives the optimizer time to accumulate reliable statistics before stepping aggressively.</i>

<b>Q2:</b> What is the key difference between cosine annealing and step decay? → <i>Cosine annealing decays smoothly and continuously, reducing sharp loss spikes at decay boundaries; step decay drops the LR abruptly at fixed intervals, which can cause sudden gradient magnitude changes.</i>

<b>Q3:</b> In fine-tuning a pretrained LLM, how would you adjust the learning rate schedule compared to pretraining from scratch? → <i>Use a much smaller peak LR (often 10x–100x lower), shorter or no warmup, and aggressive decay — because pretrained weights are already near a good optimum and large updates risk catastrophic forgetting.</i>
